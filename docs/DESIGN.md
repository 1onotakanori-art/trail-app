# TRAIL - チーム業務管理Webアプリ 全体設計書
## Team Reporting And Integrated Log

> **ステータス: v4（設計完了）** — 全機能仕様確定、実装開始準備完了

## 1. システム概要

### 1.1 コンセプト
Obsidian共有Vaultを中心としたチーム業務管理エコシステム。
報連相チャット・日程管理（ガントチャート）・Obsidian連携を1つのWebアプリに統合し、
すべての業務データをローカルに蓄積、将来的なLLM活用の基盤とする。

### 1.2 背景
- 部署全体で初となるObsidianベースのナレッジ共有・チーム連携の取り組み
- 5人の若手チーム（25〜33歳）が発信地となり、成功すれば部署全体に展開予定
- Teams/Excel for Webでは業務ログの蓄積・LLM活用が困難なため、自作アプリで代替

### 1.3 基本情報
| 項目 | 内容 |
|------|------|
| チーム規模 | 5名（管理者1名 + メンバー4名） |
| 同時接続数 | 最大5 |
| 動作環境 | 共有PC上のローカルWebアプリ（LAN内アクセス） |
| データ保存 | すべてローカル（SQLite + Obsidian Vault） |

### 1.4 現状の業務ツール構成
| 用途 | 現状 | TRAIL導入後 |
|------|------|------------|
| 業務ナレッジ | Obsidian共有Vault | Obsidian共有Vault（継続） |
| 業務連絡（報連相） | Teams | TRAIL チャット機能 |
| 日程管理 | Excel for Web | TRAIL ガントチャート |
| 作業ファイル | Box | Box（継続、TRAILからリンク） |

## 2. 技術スタック

| レイヤー | 技術 | 備考 |
|----------|------|------|
| **Frontend** | React (Vite + TypeScript) | SPA構成 |
| **Backend** | FastAPI (Python) | REST API + WebSocket |
| **DB** | SQLite | 単一ファイル、バックアップ容易。検索にFTS5使用 |
| **リアルタイム通信** | WebSocket (FastAPI内蔵) | チャット + 通知 |
| **ファイル監視** | watchdog (Python) + 手動同期ボタン | ハイブリッド方式 |
| **認証** | JWT (30日有効) + bcrypt | ブラウザlocalStorage保存 |
| **ガントチャート** | 仕様確定後にライブラリ選定 | dhtmlx-gantt / D3.js / カスタム等 |
| **Markdownレンダリング** | unified/remark 系 or markdown-it | Obsidianプレビュー用 |
| **エクスポート** | html2canvas + jsPDF | ガントPNG/PDF出力 |
| **LLM（将来）** | LM Studio | OpenAI互換APIエンドポイント |
| **プロセス管理** | バッチファイル起動 | Docker不使用 |

## 5. 業務フロー詳細

### ① 業務開始
```
[ユーザー操作]
  プロジェクト管理タブ、またはヘッダーのクイック登録(＋ボタン)で入力:
    - 業務名
    - 担当者
    - 開始日・終了日
    - タグ（複数選択可）
    - Box URL / BoxDriveパス
  ※ クイック登録は最小限の入力で即登録、詳細はプロジェクト管理タブで後から編集可能

[システム自動処理]
  1. projects テーブルにレコード作成
     → ID自動採番: P-YYYYMMDD-NNN
  2. Obsidian Vault に以下を生成:
     → 200_Projects/YYYYMMDD_業務名/
     → 200_Projects/YYYYMMDD_業務名/★業務名.md
        (400_Template/project.md をコピー、frontmatter自動記入)
  3. channels に業務専用スレッド作成
  4. channel_subscriptions に担当者と管理者を自動登録
  5. notifications に全員向け通知作成
  6. WebSocket で全員に新規業務通知 → 通知ベルに反映
```

### ② 日程計画
```
[ユーザー操作]
  - マイルストーン登録（打ち合わせ日、進捗報告日など）
  - 子タスク登録（具体的な実施事項と計画期間）
  - 子タスク間の依存関係をドラッグで矢印接続
  - ドラッグ&ドロップで日程調整

[表示]
  ガントチャートに親タスク（業務）＋子タスク＋マイルストーン＋依存矢印を表示
```

### ③ 業務遂行（Obsidian側）
```
[ユーザー操作]
  Obsidian で note テンプレートから新規ノート作成:
  ---
  date: 2026-03-21
  project: "[[★バッテリーCAE解析]]"
  owner: Takanori
  tags: [note]
  ---
  # 本日の作業内容
  ...

[システム自動処理（watchdog）]
  1. 200_Projects/ 配下のファイル変更を検知
  2. frontmatter を解析
  3. tags に "note" を含むファイルを検出
  4. ファイルパスからプロジェクトフォルダを特定
     → そのフォルダの★ファイルの project_id を取得
  5. note_sync_log に登録
  6. daily_logs にエントリ追加（obsidian_uri 自動生成）
  7. search_index に追加（全文検索用）
  8. WebSocket でフロントに更新通知
     → Vaultエクスプローラーのツリーもリアルタイム更新
```

### ④ 報連相
```
[ユーザー操作]
  - 業務スレッドまたは全体スレッドでメッセージ投稿
  - タグ選択: 報告 / 連絡 / 相談
  - @メンション
  - Obsidianリンク挿入（ファイル選択UI → URI自動生成）
    → リンク先のnote内容がプレビューウィンドウで閲覧可能
  - リアクション（絵文字）

[システム処理]
  1. messages に保存
  2. search_index に追加（全文検索用）
  3. WebSocket で該当チャンネル購読者に通知
  4. メンションされたユーザーには notifications レコード作成 + 優先通知
  5. 通知ベルのバッジ数更新
```

### ⑤ 進捗反映
```
[自動処理]
  watchdog経由で note → note_sync_log → daily_logs に自動登録
  （フォルダベースでプロジェクト紐づけ）

[ユーザー操作]
  ■ 親タスク（プロジェクト）レベル:
    - daily_logs の一言コメント手入力（任意）
    - または「LLM要約」ボタン → LM Studio で note から一言生成（任意）
    - State / Feeling 更新（任意タイミング、ワンクリック）

  ■ 子タスクレベル:
    - ガントチャート上のマス目にマウスオーバー → 吹き出し表示
    - 進捗度スライダーで変更（0% / 20% / 40% / 60% / 80% / 100%）
    - デフォルトは100%（想定通り）、変更時のみDB記録

  ■ 週次サマリー:
    - 業務展開時にガント下部に表示
    - 手動入力 or LLM生成ボタンで自動要約
    - 上司報告・週報にそのまま使用可能

[管理者向け自動フォロー（日次バッチ）]
  1. 各プロジェクトの直近 note 更新頻度チェック
  2. 以下の条件で notifications + ダッシュボードにアラート表示:
     - 2日以上 note 更新なし（State=進行中のもの）
     - Feeling が「遅延しそう」「相談したい」
     - [将来] LLMによる進捗分析
```

### ⑥ 業務クローズ
```
[ユーザー操作]
  プロジェクト管理タブまたは業務詳細画面で:
  - 最終サマリー入力（テキスト）
  - 最終アウトプット登録（Box URL / Obsidianリンク / 複数可）
  - 「クローズ」ボタン押下

[システム自動処理]
  1. projects.state → "完了", closed_at 記録
  2. projects.archived → true
  3. Obsidian ★業務名.md に追記:
     - 完了日
     - 最終サマリー
     - アウトプットリンク一覧
  4. ガントチャート上で完了色表示
  5. チャットスレッドは読み取り専用に
  6. ダッシュボードのアクティブ一覧から除外（アーカイブ表示で閲覧可）
```

## 6. ガントチャート詳細仕様

### 6.1 表示モード

| モード | 内容 | 主な用途 |
|--------|------|----------|
| **全体表示** | 全メンバーの親タスクを一覧。子タスクは折りたたみ。 | 上司報告、チーム全体の俯瞰 |
| **個人表示** | 特定メンバーの親タスク + 子タスク展開可能 | 個人の詳細進捗確認、上司報告 |
| **タグ別表示** | タグでフィルタして横断的に表示 | 技術領域別の横断確認 |

### 6.2 表示構造（全体表示）

```
┌──────────────────────────────┬─ 3/17 ─┬─ 3/18 ─┬─ 3/19 ─┬─ 3/20 ─┬─ 3/21 ─┐
│ ▶ バッテリーCAE    🔵 ✓ 💬  │ ●░░░░░ │ ●░░░░░ │ ●░░░░░ │ ●░░░░░ │ ░░░░░░ │
│ ▶ 電極設計         🔵 △ 💬  │        │ ●░░░░░ │ ●░░░░░ │ ●░░░░░ │ ●░░░░░ │
│ ▶ 報告書作成       ⚪ 💬 💬 │        │        │        │ ░░░░░░ │ ░░░░░░ │
└──────────────────────────────┴────────┴────────┴────────┴────────┴────────┘

タスク名の右:  State(●アイコン) + Feeling(テキストアイコン) + 💬(チャットボタン)
親タスク行:    ● = noteが存在する日（実績ドット）
               ░ = 子タスクの計画範囲（薄色バー、重ね合わせ）
```

### 6.4 親タスク行の表示ロジック

- **計画バー:** 子タスクの planned_start〜planned_end の最小〜最大範囲。子タスク未登録時は親の start_date〜end_date。
- **実績ドット（●）:** daily_logs にその日・そのプロジェクトのレコードが存在する日。
- **State/Feeling:** タスク名の右に常時表示。クリックで変更UI。
- **チャットボタン（💬）:** クリックで右側にチャットパネルをスライドイン表示。

### 6.5 子タスク行の表示ロジック

- **計画バー（薄色）:** planned_start〜planned_end の期間。
- **実績表示（進捗度）:** 計画期間内の当日以降の日付に対して表示。
  - デフォルト: 100%（想定通り進められた = 青塗りつぶし）
  - ユーザーが変更した場合: task_daily_progress のレコード値で描画
- **未到達日（将来日程）:** 計画色のまま。
- **ステータス:** actual_start は status=進行中 に変更した日に自動記録。actual_end は status=完了 に変更した日に自動記録。

### 6.6 進捗度の視覚表現

| 進捗度 | 表現 | 説明 |
|--------|------|------|
| 100% | 青塗りつぶし | 想定通り（デフォルト） |
| 80% | 密ドット | おおむね進んだ |
| 60% | 中ドット | 半分以上は進んだ |
| 40% | 粗ドット | やや遅れ |
| 20% | 極粗ドット | かなり遅れ |
| 0% | グレー塗り | 未実施 |

### 6.7 依存関係の矢印

- **作成:** 子タスクのバー右端（最終日）からドラッグ → 別の子タスクのバー左端（開始日）にドロップ。
- **表示:** 先行タスクの最終日の右端から後続タスクの開始日の左端へSVG pathで矢印描画。
- **削除:** 矢印をクリック → 削除ボタン表示、または右クリックメニュー。
- **制約:** 同一プロジェクト内の子タスク間のみ。
- **タイプ:** FS（Finish-to-Start）のみ。将来的にSS/FF/SF追加可。

## 8. Obsidian連携仕様

### 8.1 Vault構造（アプリが管理する部分）

```
SharedVault/
├── 200_Projects/
│   └── YYYYMMDD_業務名/
│       ├── ★業務名.md          ← 中核ファイル（自動生成・自動更新）
│       └── *.md                 ← noteファイル（ユーザーが手動作成）
├── 400_Template/
│   ├── project.md               ← プロジェクトテンプレート
│   └── note.md                  ← noteテンプレート
└── _app/                        ← アプリ専用出力領域
    ├── daily-digest/            ← 日次ダイジェスト（将来）
    └── schedule/                ← スケジュールMD（将来）
```

### 8.2 note.md テンプレート（frontmatter）

```yaml
---
date: {{date}}
project: "[[★{{project_name}}]]"
owner: {{owner_name}}
tags: [note]
---
```

- `date`: 自動入力（ノート作成日）
- `project`: ★付き中核ファイルへのObsidianリンク（グラフビューで関係可視化）
- `owner`: note作成者（担当者）の名前
- `tags`: `#note` は自動付与
- **project_id は記載しない**（フォルダベースで判定するため）

### 8.3 project.md テンプレート（★ファイル）

```yaml
---
tags: [project]
project_id: {{project_id}}
project_name: {{project_name}}
owner: {{owner_name}}
start_date: {{start_date}}
end_date: {{end_date}}
state: 進行中
box_url: {{box_url}}
box_local_path: {{box_local_path}}
---

# {{project_name}}

## 概要
（ここに業務概要を記載）

## 関連リンク
- Box: [作業フォルダ]({{box_url}})

## 完了情報
<!-- クローズ時に自動追記 -->
```

### 8.4 Obsidian URI 生成ルール

```
obsidian://open?vault={VAULT_NAME}&file={ENCODED_RELATIVE_PATH}
```

- VAULT_NAME: システム設定で登録
- ENCODED_RELATIVE_PATH: Vault ルートからの相対パスをURIエンコード
- アプリ側でプロジェクトID → フォルダパス → URI を自動生成可能

### 8.5 watchdog 監視ルール

- **監視対象:** `200_Projects/` 配下の `.md` ファイル
- **トリガー:** ファイル作成 / 変更
- **プロジェクト判定:** ファイルパスからプロジェクトフォルダを特定 → そのフォルダの★ファイルの project_id を参照
- **処理対象:** frontmatter の tags に `note` を含むファイル
- **除外:** `★` で始まるファイル（中核ファイルはアプリ側が管理）
- **フォールバック:** 手動同期ボタン（UIに配置）でオンデマンドスキャン

### 8.6 watchdog → daily_logs 登録フロー

```
1. ファイル変更検知
2. ファイルパスからプロジェクトフォルダを特定
   例: 200_Projects/20260321_バッテリーCAE/メッシュ検討.md
       → プロジェクトフォルダ: 20260321_バッテリーCAE
3. ★ファイル（★バッテリーCAE.md）のfrontmatterからproject_idを取得
4. noteのfrontmatterからdateを取得
5. note_sync_log にレコード作成（file_hashで重複排除）
6. daily_logs にレコード作成（project_id + date + note_path）
7. obsidian_uri を自動生成
8. search_index に追加
9. WebSocketで接続中のクライアントに note_synced イベント送信
   → Vaultエクスプローラーのツリーもリアルタイム更新
```

### 8.7 Obsidianプレビュー機能

チャット内のObsidianリンク、およびVaultエクスプローラー上のファイルからnote内容を閲覧可能。

**バックエンド:**
- `/api/vault/preview/{note_path}` エンドポイント
- Vault内のMarkdownファイルを読み込み
- Markdownをレンダリング用HTMLに変換（unified/remark または markdown-it）
- Obsidian内部リンク（`[[...]]`）を解決してリンク化
- 画像パスをVaultルートからの相対パスで解決

**フロントエンド:**
- フローティングウィンドウ（モーダル）でHTMLを表示
- ウィンドウ内スクロールで全文閲覧可能
- ドラッグで移動、リサイズ可能
- 🔗ボタン: Obsidian URI schemeでアプリを直接起動
- ×ボタン: ウィンドウを閉じる

**使用箇所:**
- チャットタブ: Obsidianリンクをクリック → プレビューウィンドウ
- メインタブ: Vaultエクスプローラー上のファイルにマウスオーバー → 吹き出しプレビュー
- メインタブ: ガントの親タスク吹き出し内のnoteリンク → プレビューウィンドウ
- 共通のレンダリングエンジンを全箇所で共有
